# Конкурентная запись в map (Go)
```map``` в Go не потокобезопасен: при записи из нескольких горутин без синхронизации возможны data race и ```fatal error: concurrent map writes.```

В решении используется ```sync.RWMutex```: запись через ```Lock```, чтение через ```RLock```, поэтому доступ к ```map``` безопасен при конкуренции (Go 1.16).

Проверка на гонки:

```bash
go run -race main.go
```
